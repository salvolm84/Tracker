# Tracker

Desktop activity tracker built with Tauri, React, and Mantine.

## What It Does

- Collects activity records through a structured multi-user form.
- Loads owners, projects, departments, and categories from editable text resource files.
- Persists each submission into one shared JSON database file.
- Stores uploaded attachment bytes in a managed attachment folder beside the JSON database, keeping the database focused on metadata.
- Uses a file lock during writes to reduce corruption risk when multiple app instances save at the same time.

## Prerequisites

Install these once before building or running the app:

- Node.js 22 or newer.
- Rust stable from <https://rustup.rs/>.
- Tauri platform prerequisites for your OS.

macOS also needs Xcode Command Line Tools:

```bash
xcode-select --install
```

Windows also needs Microsoft WebView2 and Microsoft C++ Build Tools. The simplest path is installing Visual Studio 2022 Build Tools with the "Desktop development with C++" workload.

## Install Dependencies

Run this from the repo root on both macOS and Windows:

```bash
npm install
```

For CI or a clean reproducible install, use:

```bash
npm ci
```

## Run In Development

macOS:

```bash
cd /Users/salvolm/Tracker
npm run tauri dev
```

Windows PowerShell:

```powershell
cd C:\path\to\Tracker
npm run tauri dev
```

The dev command starts Vite on `http://localhost:1420/` and opens the native Tauri desktop shell.

## Build A Runnable App Locally

macOS:

```bash
npm install
npm run package:mac
```

Windows PowerShell:

```powershell
npm install
npm run package:windows
```

Both commands produce runnable installers or app bundles under:

```text
src-tauri/target/release/bundle/
```

Expected macOS outputs include a `.dmg` installer and app bundle. Expected Windows outputs include an `.msi` and/or `.exe` installer, depending on the Tauri bundler target available on that runner.

The `package:mac` script intentionally builds the `.app` bundle only. The GitHub release workflow zips that `.app` bundle so it can be downloaded and run without depending on local DMG packaging tools.

## GitHub Release Build

This repo includes a GitHub Actions workflow at `.github/workflows/release.yml`.

Pushing a tag like `v0.1.0` starts two native builds:

- macOS build on `macos-latest`.
- Windows build on `windows-latest`.

When both builds finish, the workflow creates a GitHub release and uploads the runnable macOS and Windows artifacts.

The macOS release artifact is a zipped `Tracker.app`. The Windows release artifacts are installer files generated by Tauri on the Windows runner.

To create a release manually from a prepared commit:

```bash
git tag -a v0.1.0 -m "Tracker v0.1.0"
git push origin main
git push origin v0.1.0
```

Then open the repository Actions tab and wait for the `Release` workflow to finish. The finished release appears under GitHub Releases.

## Shared Database Path

By default, the app stores the JSON database as a single file named `activity-db.json`
next to the app location:

- In development, next to the compiled Tauri binary.
- In a packaged macOS deployment, next to `Tracker.app` in the same folder.

If that file does not exist when the app starts, the app creates a new database file there automatically.
Attachments are stored beside the database in a folder named after the database stem, for example `activity-db.attachments`.

To point the app at a different file, set `TRACKER_DB_PATH` before launching it:

```bash
export TRACKER_DB_PATH="/Users/Shared/tracker/activity-db.json"
npm run tauri dev
```

For Windows deployment later, you can set this to a shared network path such as `\\server\share\activity-db.json`.
